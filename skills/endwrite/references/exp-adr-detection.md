# EXP/ADR 후보 감지 상세 로직

endwrite step 3 에서 참조.

## 감지 범위 (좁게 잡기)

다음 중 **하나 이상** 해당할 때만 후보로 올림:
- 세션에서 명시적으로 "실험해봤다", "비교했다", "성능 측정", "A/B", "벤치마크" 결과가 등장
- "~로 결정했다", "~ 선택", "~ 쓰기로" 가 명시적으로 나온 단락
- 사용자가 "ADR 만들자", "EXP로 남기자" 같은 의도 표현

모호한 "고민했다", "생각해봤다" 수준은 제외.

## 중복 체크 (효율)

각 후보에 대해:
1. 키워드 2~3개 뽑기 (제목 슬러그 후보).
2. `Grep "<키워드>" --glob "$BRAIN/01_projects/*/experiments/*.md" -l` 로 이미 있는 파일 체크.
3. 매칭되면: 세션 로그 "변경된 파일" 에 기존 `[[EXP-NNN]]` wikilink 추가. 새 파일 만들지 않음.

## 사용자 확인 포맷

```
💡 새로 기록할 후보:
1. [실험] "<한 줄 요약>" → EXP-NNN로 저장? (y/n/수정)
2. [결정] "<한 줄 요약>" → ADR-NNN로 저장? (y/n/수정)
```

## 저장 로직

**y** 응답:
- 번호: `ls $BRAIN/01_projects/<PROJECT>/experiments/EXP-*.md 2>/dev/null | tail -1` → 다음 번호 3자리 zero-pad.
- 템플릿: `$BRAIN/03_templates/experiment.md` 또는 `adr.md`.
- 치환: `{{EXP_ID}}`/`{{ADR_ID}}`, `{{TITLE}}`, `{{DATE}}`.
- 본문 prefill: 세션에서 추출한 가설/결정 내용을 해당 섹션에 미리 채우기.
- 저장 경로: `$BRAIN/01_projects/<PROJECT>/(experiments|decisions)/<ID>-<slug>.md`.

**수정** 응답:
- 제목/슬러그만 사용자 입력 받고 위와 동일 저장.

**n** 응답:
- 파일 만들지 않음.
- 세션 로그 "다음 세션으로 넘기는 컨텍스트" 에 `- 기록 후보: <요약>` append (잊지 않게).

## 프로젝트 지정

- `01_projects/`에 하위가 하나뿐이면 자동 선택.
- 여럿이면 사용자에게 "어느 프로젝트인가요?" 물어봄.

## 안전장치

- 세션당 최대 2개.
- 키워드 겹침 ≥2 이면 "이미 있는 것 같은데 같은 거 아닌가요?" 한 번 확인.
- 대화가 짧거나(>10턴 미만) 코드 변경 없으면 스킵.
